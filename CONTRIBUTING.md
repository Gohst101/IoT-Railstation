# Contributing to IoT-Railstation

First of all, thank you for taking the time to contribute to **IoT-Railstation**! 🚂

This project is meant to grow through ideas, experiments, improvements, and community collaboration.  
Whether you want to improve code, suggest new hardware features, fix bugs, or help with documentation - every contribution is welcome.

---

## Ways to Contribute

You can contribute in many different ways, including:

- **Reporting bugs** – **Important:** if the issue poses a security risk, please contact us directly via email: [IoT-Railstation-security@ghost-exe.de](mailto:IoT-Railstation-security@ghost-exe.de)
- **Suggesting new features**
- **Improving documentation**
- **Optimizing code**
- **Adding support for new hardware**
- **Improving MQTT communication**
- **Sharing ideas for automation or station features**

---

## Before You Start

Before working on a larger change, please consider opening an **Issue** first.

This helps to:
- **discuss the idea**
- **avoid duplicate work**
- **make sure the feature fits the project goals**

Small fixes and documentation improvements can usually be submitted directly.

---

## How to Contribute

### 1. Fork the Repository
Create your own copy of the repository.

### 2. Create a Branch
Use a clear branch name, for example:

- `feature/mqtt-reconnect-improvement`
- `fix/wifi-timeout-bug`
- `docs/setup-guide-update`

### 3. Make Your Changes
Keep your changes as clean and focused as possible.

### 4. Test Your Changes
If possible, test your changes before submitting them.

Examples:
- Does the ESP32 still compile?
- Does MQTT still connect properly?
- Does the Web Server work?

### 5. Open a Pull Request
Submit a Pull Request with a short but clear explanation of what you changed and why.

---

## Code Style

Please try to keep the code:

- readable
- modular
- well-structured
- easy to maintain

### General Guidelines
- Use meaningful variable and function names
- Avoid unnecessary complexity
- Add comments where needed
- Keep hardware-related code understandable
- Write code in a way that others can expand later

---

## Documentation Contributions

Documentation improvements are highly appreciated.

This includes:
- setup guides
- hardware explanations
- MQTT topic documentation
- wiring information
- troubleshooting steps

Clear documentation helps the project grow faster and makes it easier for others to join.

---

## Hardware Contributions

If you want to contribute hardware-related ideas or changes, please include as much detail as possible.

Helpful things to include:
- pin usage
- wiring diagrams
- module names
- sensor information
- expected behavior

---

## Feature Suggestions

Feature suggestions are welcome, especially if they fit the project vision of:

- smart railway automation
- interactive station systems
- modular IoT components
- connected toy train control

If you have an idea, feel free to open an **Issue** and describe it.

---

## Project Vision

IoT-Railstation is intended to become an open platform for building smarter and more interactive toy railway systems.

The goal is not just to create one finished setup, but to build a system that others can improve, expand, and experiment with.

---

## Be Respectful

Please be respectful and constructive when contributing.

This project is built for learning, collaboration, and technical creativity.

---

## Questions?

If you are unsure whether something fits the project, feel free to open an **Issue** and ask.

Thanks for contributing to **IoT-Railstation** 🚂
